01 · Setting the Stage

AI promises acceleration — but acceleration without structure creates chaos

The biggest hidden cost of AI in product development today isn't model usage. It's translation tax.

Every time work moves between disciplines — PM to research, research to design, design to architecture, architecture to development — context gets compressed. Assumptions mutate. Nuance disappears.

With AI, this problem compounds:

  • Prompts summarize instead of preserve.
  • Outputs vary depending on framing.
  • Teams regenerate work because prior reasoning wasn't inherited.
  • Compliance is reviewed after decisions are made.
Translation tax costs time, money, and risk exposure.
AI can reduce that gap — but not without structure.

WAVE (Workflow for Agentic Value Engineering) was created to operationalize AI inside enterprise guardrails.

05 · The WAVE System

Structured stages with deterministic handoffs

WAVE moves through structured stages: PM → Research → Design → Architecture → Development.

Each stage:

  • Inherits upstream structured artifacts
  • Produces schema-bound outputs
  • Passes forward deterministic context
  • Requires a hard human approval gate

AI accelerates structure. Humans authorize movement.

Stage 1

PM Agent

The PM Agent generates a structured PRD.md using enforced templates.

Templates require:

  • Problem framing
  • Constraints
  • Risk tier
  • Compliance notes
  • Success metrics

Outputs are structured, not narrative. Upstream clarity reduces downstream entropy.

Stage 2

Research Agent

The Research Agent:

  • Classifies work as evaluative or generative
  • Applies a 10-question structured framework
  • Identifies assumption risk
  • Flags regulatory exposure
  • Produces a structured research brief
  • Consolidates findings into research.md

Research becomes executable input — not slideware.

Stage 3

Design Agent

The Design layer is an orchestrated system, not a single agent.

Design Orchestrator Agent

The orchestrator receives PRD.md + research.md, generates design.md, then coordinates the full pipeline:

Route to sub-agents Human communication Design gate enforcement Epic & story generation

It coordinates — it does not operate alone.

Design Sub-Agents

Sub-agents specialize in:

designSystem.md Wireframes State modeling Token mapping Intent modeling Constraint validation Version control

Structured outputs include system model definitions, guardrails, state transitions, token usage, explicit constraints, and version logs.

Design becomes deterministic reasoning — not aesthetic interpretation.

Once the design gate is approved by a human, the workflow advances.

Stage 4

Architecture Agent

Architecture inherits design.md directly. It:

  • Formalizes data models
  • Encodes validation logic
  • Defines compliance hooks
  • Generates epics and stories
  • Establishes audit logging requirements

No reinterpretation. No context loss.

Stage 5

Development Agent

The Development Agent converts structured artifacts into executable work:

  • Epics
  • Stories
  • Acceptance criteria

Developers inject upstream artifacts into their IDE. They inherit reasoning — not summaries.

Translation tax collapses.

07 · WAVE in Practice

AI Transaction Summaries — a fintech-relevant example

To illustrate how WAVE works in a fintech-relevant context: a product team wants to launch AI-generated transaction summaries.

Without structure, this introduces risk:

  • Model hallucination
  • Misrepresentation of financial data
  • Regulatory scrutiny
  • Trust degradation
PM

Define constraints in PRD.md

  • Problem framing for transaction summaries
  • Risk tier classification
  • Compliance requirements scoped
Research

Flag hallucination & compliance risks

  • Evaluative framework applied
  • Regulatory exposure identified
  • Assumption risk documented
Design

Encode guardrails into design.md

  • Confidence thresholds defined
  • Schema-bound outputs enforced
  • Fallback logic specified

Architecture enforces validation rules. Development inherits deterministic acceptance criteria.

AI remains bounded. Risk is surfaced early. Compliance is proactive.
Execution becomes structured, not reactive.
